# Research-Flow 用户需求文档

## 1. 文档说明

### 1.1 编写目的

本文档旨在完整、清晰地描述 Research-Flow 平台的用户需求，作为后续系统设计、功能开发与验收测试的基准依据。

### 1.2 适用范围

本文档适用于 Research-Flow 项目的所有参与方，包括产品设计、开发实现与最终用户。

### 1.3 术语定义

| 术语 | 说明                      |
|------|-------------------------|
| 论文入库 | 将一篇论文的 PDF 及元数据纳入平台统一管理 |
| 自动分析文档 | 由系统根据模板自动生成的论文分析内容      |
| 人工笔记 | 用户手动撰写的个人理解与补充内容        |
| Zotero MCP | 用于读取 Zotero 文献库并与平台建立集成的 MCP 连接能力 |
| 观点 | 作者针对该领域特定研究问题的核心主张      |
| 课题 | 用户正在研究或已归档的一个具体研究项目     |
| Skills | 平台内可复用的自动化分析/生成能力单元     |
| 知识图谱 | 论文、数据集、课题之间关系的结构化图状表示   |
| Rebuttal | 针对审稿人意见的书面回复            |

## 2. 项目背景与目标

### 2.1 背景

科研人员在研究过程中面临以下痛点：

- 文献数量庞大，手动管理效率低、易遗漏
- 论文分析缺乏统一结构，难以横向对比
- 论文观点分散，难以形成系统的知识积累，写作时难以快速提取和复用
- 实验规划缺少系统性参考，依赖个人经验
- 课题进展分散在多处文档，缺乏整体视图
- 组会汇报材料准备依赖手动整理，重复整理成本较高，且难以复用已有分析结果
- 论文写作耗时耗力，缺少结构化辅助
- 会议截止时间分散，容易错过重要节点

### 2.2 目标

构建一个面向科研人员的**全生命周期研究工作流管理平台**，覆盖从文献调研、组会汇报到论文录用的完整流程，通过自动化辅助与人工审核相结合，降低科研管理成本，提升研究效率。

### 2.3 核心工作流

```
文献调研分析 → 科研进展追踪 → 实验规划 → 实验结果分析 → 组会汇报准备 → 论文写作 → 投稿/Rebuttal → 录用后处理
```

## 3. 用户角色

### 3.1 主要用户：科研人员

- 在读研究生、博士生、科研工作者
- 需要管理大量文献、维护多个研究课题
- 有阶段汇报、论文写作与投稿需求
- 希望减少重复性整理工作，专注于研究本身

### 3.2 用户核心诉求

| 诉求 | 说明 |
|------|------|
| 高效入库 | 快速将目标论文纳入管理，不打断阅读节奏 |
| 结构化分析 | 每篇论文有统一格式的分析摘要，便于对比 |
| 观点提炼 | 从论文中抽取核心观点，形成系统积累 |
| 知识沉淀 | 个人理解与系统分析共存，长期积累可复用 |
| 课题追踪 | 清晰掌握每个课题的进展与关联文献 |
| 汇报辅助 | 基于课题与论文积累快速生成组会汇报材料，减少重复整理 |
| 写作辅助 | 在已有积累基础上快速生成论文初稿 |
| 信息聚合 | 会议时间、每日新论文等信息集中管理 |

## 4. 功能需求

### 4.1 文献管理

#### 4.1.1 文献入库

**需求描述：**  
用户可通过多种方式将论文纳入平台管理。

**详细需求：**

- **批量检索入库**：用户输入关键词，系统自动搜索相关论文，展示搜索结果列表，用户确认后批量入库
- **单篇手动入库**：用户通过 DOI、论文名称或 URL 检索单篇论文并入库
- **PDF 获取**：入库时自动关联或获取论文 PDF 文件
- **分类存储**：论文按用户定义的「大方向 → 子方向」层级结构归类，入库时可指定归属分类
- **元数据记录**：自动记录论文标题、作者、发表年份、来源会议/期刊等基本信息

**验收标准：**

- 关键词搜索结果准确相关，支持预览后选择性入库
- 手动入库流程不超过 3 步操作
- 入库后 PDF 可在平台内打开查看

---

#### 4.1.2 文献分析

**需求描述：**  
论文入库后，系统自动生成结构化分析文档，同时为用户提供独立的人工笔记空间。

**详细需求：**

- **自动分析文档**：
  - 论文入库后自动触发生成
  - 分析内容由预设模板约束，输出结构统一
  - 模板内容包含但不限于：研究问题、核心方法、主要贡献、局限性、实验结论
  - 用户可自定义或修改分析模板
- **人工笔记文档**：
  - 每篇论文配套一份独立的人工笔记文件
  - 提供笔记模板作为参考，用户可自由编辑
  - 与自动分析文档相互独立，互不覆盖
- **数据集与实验信息提取**：
  - 自动从论文中识别并提取所用数据集名称
  - 提取对应的实验设置（评估指标、对比基线方法等）
  - 提取该数据集上的实验结果数值

**验收标准：**

- 自动分析文档在入库后自动生成，内容符合模板结构
- 人工笔记与自动分析文档并列存在，修改互不影响
- 数据集及实验结果提取准确率达到可用水平

---

#### 4.1.3 论文推荐

**需求描述：**  
系统根据用户已有的研究方向和已入库论文，主动推荐相关有价值的论文。

**详细需求：**

- 基于当前入库论文内容，推荐相关未入库论文
- 推荐结果包含论文基本信息及推荐理由
- 用户可一键将推荐论文入库

---

#### 4.1.4 Zotero 集成

**需求描述：**  
平台需支持与用户现有 `Zotero` 文献库建立连接，复用已有收藏、元数据与附件组织结果，减少重复入库与重复整理成本。

**详细需求：**

- 平台可通过 `Zotero MCP` 读取用户的 `Zotero` 文献库、Collection 结构、条目元数据与附件信息。
- 支持选择指定 Collection、标签或单篇条目导入到平台。
- 支持导入论文标题、作者、年份、会议 / 期刊、DOI、URL、摘要、标签等基础信息。
- 支持识别并关联 `Zotero` 中已有的 PDF 或附件信息，减少重复下载。
- 导入完成后可直接接入平台现有的分析、分类、课题关联与写作辅助流程。

**验收标准：**

- 用户可在平台内成功连接 `Zotero MCP`，并浏览可导入的 `Zotero` 条目或 Collection。
- 用户可选择性导入论文，导入后信息完整可用，并可追溯至对应 `Zotero` 条目。

---

### 4.2 每日论文推送

**需求描述：**  
系统每日自动推送领域内最新论文，供用户快速筛选，避免错过前沿进展。

**详细需求：**

- 系统每日定时自动抓取 `Arxiv` 最新发布的论文
- 对每篇论文自动生成摘要分析与价值评分
- 推送结果以列表形式呈现，用户可快速浏览标题与摘要
- 用户筛选操作：
  - **入库**：将选中论文正式纳入论文库，并触发自动分析文档生成
  - **忽略**：不选择的论文次日自动清除，不占用存储空间
- 支持按评分、研究方向等维度对推送列表进行筛选排序

**验收标准：**

- 每日推送在固定时间段内完成更新
- 未入库论文在次日推送更新后自动清除
- 用户可在 5 分钟内完成一批推送论文的浏览与筛选决策

---

### 4.3 数据集管理

**需求描述：**  
独立管理研究领域内的数据集信息，并与论文形成关联。

**详细需求：**

- 每个数据集维护一份独立档案，包含：
  - 数据集基本介绍（名称、规模、任务类型、获取方式等）
  - 使用该数据集的已入库论文列表
  - 各论文在该数据集上的实验结果汇总对比
- 数据集档案可手动创建，也可在论文分析时自动生成/关联
- 支持通过数据集查找所有相关论文
- 支持通过论文查看其所用的全部数据集

**验收标准：**

- 数据集与论文之间的关联关系准确
- 在数据集页面可直接横向对比不同论文的实验结果

---

### 4.4 结构化知识提炼

从已入库论文中自动提取结构化知识，并按类型进行分离管理，形成可检索、可关联的领域知识库。知识类型被拆分为“观点类”与“定义特征类”两大模块：

#### 4.4.1 观点提炼（用于引用的主观论断）

**需求描述：**
观点并非对整篇文章的摘要总结，而是专门提取**论文作者针对当前领域、现有方法或特定问题发表的明确看法、批判或主张**。这些观点应具有强烈的“可引用性”（Citable），方便用户在撰写自己论文的 Introduction 或 Related Work 时直接作为引用素材。

**详细需求：**
- 每条观点必须是作者的一个明确论断（Claim），而非中性的事实说明。需按以下逻辑分类归档：
  - **领域现状评述**：作者对当前研究趋势、现实需求或该领域重要性的主观评价（如“XX 是当前阻碍落地的最大瓶颈”）。
  - **对现有方法的批判**：作者对前人工作指出的具体缺陷或抨击（如“现有方法在 XX 场景下会失效，因为忽略了 XX 因素”）。
  - **核心启发/主张**：作者在方法论层面提出的核心假说或洞见（如“我们认为，引入 XX 机制才能根本上解决 XX 问题”）。
  - **关键性结论**：打破常规认知或具有强主导性的实验发现（如“实验证明，XX 特征在提升鲁棒性上比 XX 更具决定性”）。
- 结构化字段需包含：**观点标题、观点类型、中文简要概括、英文原文依据（必填，确保引用的准确与原汁原味）**。
- **引用对象溯源（极其重要）**：当作者的批判或评述明确指向某篇参考文献时（如“相比于 [12] 提出的方法...”），系统必须准确识别并提取该被引用对象，标注在结构化字段中，形成 `[本文] -> 批判/评价 -> [参考文献]` 的关联网络。
- 用户可对自动生成的观点进行编辑、合并、删除、重分类和补充备注。

#### 4.4.2 领域定义与特征提取

**需求描述：**
从论文中结构化地提取客观的、具有复用价值的领域名词、数学定义、特征描述及评估标准，辅助用户快速扫盲和搭建代码工程。

**详细需求：**
- 系统需自动识别并提取以下类型的客观定义信息：
  - **核心概念/术语定义**：论文中重点介绍的领域专有名词及其学术解释。
  - **任务定义类**：说明某类研究任务的学术定义、目标、输入规范、输出形式及边界条件。
  - **特征定义类**：说明特定领域内关键特征的物理含义、多模态语义或数学公式表达。
  - **特征提取类**：说明提取上述特征所采用的具体算法、网络结构（如 Transformer Encoder 结构设计）或数据处理流程。
  - **评估指标定义**：论文中使用的性能评估指标的数学计算方式或度量逻辑。
  - **数据构建/标注规则**：针对数据集类论文，提取其数据采集、数据清洗或人工标注的具体规范与准则。
- 结构化字段需包含：**定义标题、定义类型、中文详细解释/公式描述/流程总结**（*注：该类别的重点在于对客观知识的准确归纳与解释，不强制要求保留“英文原文依据”字段*）。
- 用户可对提取的定义进行勘误、补充笔记或归类。

**验收标准：**
- 可准确从论文中抽取上述两类结构化知识。
- 用户可利用观点库梳理研究脉络；利用定义库快速查找“某个任务是什么”、“某种特征怎么提取”、“指标怎么计算”。



### 4.5 实验规划

**需求描述：**  
基于当前课题在科研进展追踪中积累的文献分析与 Related Work 内容，辅助用户制定实验方案。

**详细需求：**

- 系统汇总**当前课题**已入库相关论文的实验设置信息（数据集、评估指标、对比方法、超参数等）
- 以结构化方式呈现相关工作的实验设计全景，支持横向对比
- 系统分析**当前课题**的已入库相关论文，分析其方法以及局限性，总结出可扩展的研究方向，自动生成实验规划建议草稿
- 用户可在草稿基础上修改和确认最终实验方案

**验收标准：**

- 实验规划建议来源可追溯，可关联到具体参考论文
- 用户可编辑并保存最终实验方案

### 4.6 课题管理

**需求描述：**  
提供完整的课题生命周期管理能力，支持从立项到结题的全程记录与辅助。

本模块对应核心工作流中的“科研进展追踪”阶段，为实验规划、实验结果分析、组会汇报准备与论文写作提供统一上下文。

#### 4.6.1 课题结构

每个课题包含以下六个核心模块：

**① Overview（概览）**

| 项目 | 说明 |
|------|------|
| 目标定位 | 作为课题首页，帮助用户快速掌握当前课题的研究范围、素材覆盖度与整体推进状态 |
| 自动能力 | 自动汇总当前课题的主要研究领域、关联数据集、已覆盖论文、代表性工作、最近更新与阶段性进展 |
| 人工介入 | 用户可确认研究方向标签、补充阶段目标与里程碑、标记重点论文/数据集、修正关联关系 |
| 输出形式 | 结构化概览页，支持卡片总览、结构化清单、进展时间线、关系视图等展示方式 |
| 输出内容 | 包含主要研究领域、关联数据集、论文覆盖统计、近期进展摘要、关键结论、待解决问题与下一步动作 |

**② Related Work（相关工作）**

| 项目 | 说明                                                           |
|------|--------------------------------------------------------------|
| 自动能力 | 基于课题主题，从已入库论文中识别相关文献并自动组织候选相关工作                              |
| 人工介入 | 支持用户审核范围、调整分类方式、补充遗漏论文与主观理解                                |
| 输出形式 | 结构化的相关工作综述文档，包含论文关联链接                                        |
| 输出内容 | 包含研究背景、相关工作分类、代表性论文概述、方法差异分析及优缺点总结等内容；正文按学术写作习惯插入引用标注，文末附完整参考文献信息及对应链接 |

**③ Method（方法设计）**

| 项目   | 说明 |
|------|------|
| 自动能力 | 从相关论文中自动提取现有方法的局限性，总结可扩展方向 |
| 人工介入 | 用户在此基础上撰写自身方法的 Contribution |
| 输出形式 | 包含 Limitations 分析和 Contribution 陈述的方法文档 |
| 输出内容 | 包含现有方法的主要局限、问题成因分析、可行改进方向、潜在创新切入点，以及对自身方法 Contribution 的结构化表述建议 |

**④ Experiment（实验记录）**

| 项目 | 说明                                                                                |
|------|-----------------------------------------------------------------------------------|
| 自动能力 | 基于实验规划模块自动生成实验方案；实验完成后自动分析结果                                                      |
| 人工介入 | 用户执行实验、录入实验结果数据                                                                   |
| 输出形式 | 包含实验方案、原始结果、分析结论的完整实验记录                                                           |
| 输出内容 | 文档包含主实验与消融实验的完整记录，涵盖实验目的、设计方案（数据集、评估指标、对比方法）、失败结果分析以及后续改进建议、最终结果分析等内容 |

**⑤ Conclusion（结论）**

| 项目 | 说明 |
|------|------|
| 自动能力 | 基于以上三个模块内容自动生成结论草稿 |
| 人工介入 | 用户审核并修改最终结论 |
| 输出形式 | 结构完整的结论文档 |
| 输出内容 | 包含研究总结、方法优势、实验结论、未来工作展望等内容的结论文档 |

**⑥ Manuscript（论文草稿）**

| 项目 | 说明 |
|------|------|
| 目标定位 | 作为课题阶段性产出，持续沉淀一份可迭代、可投稿的完整论文草稿 |
| 自动能力 | 基于 Related Work、Method、Experiment、Conclusion 的内容自动编排论文结构并生成全文初稿，补齐章节衔接、引用占位与术语统一 |
| 人工介入 | 用户可逐章节审核、改写、锁定已确认内容，补充图表说明、实验细节和投稿要求相关信息 |
| 输出形式 | 结构化的 Manuscript 工作区，支持分章节编辑、整篇预览与版本留存 |
| 输出内容 | 包含 Abstract、Introduction、Related Work、Method、Experiment、Conclusion 等章节的完整论文草稿；正文保留引用标注，文末自动整理参考文献信息及对应链接 |

#### 4.6.2 课题通用需求

- 支持创建、编辑、归档课题
- 课题内可关联任意已入库论文，并标注关联类型（如"相关工作"、"方法参考"等）
- 课题文档支持 Markdown 格式编辑与渲染
- 支持课题状态管理：进行中 / 已归档
- Overview 中的关键统计与近期进展在课题关联论文、数据集或实验记录更新后自动刷新
- Manuscript 与各章节草稿保持关联，重新生成时支持保留已确认内容
- 支持在课题下创建组会汇报任务，并基于关联论文与课题文档自动生成可编辑的汇报大纲及逐页内容草稿，保留来源关联信息

**验收标准：**

- 六个模块文档均可独立编辑，自动生成内容不覆盖人工内容
- 课题与论文的关联关系在知识图谱中可视化呈现
- 课题下的组会汇报任务可关联已有论文与课题文档，生成可编辑的汇报草稿，并支持来源追溯

### 4.7 论文写作

**需求描述：**  
基于课题管理中沉淀的 Related Work、Method、Experiment、Conclusion 等内容，辅助用户生成、编辑并迭代完整论文草稿。

**详细需求：**

- 针对论文各章节提供独立的写作辅助能力，包含但不限于：
  - Abstract（摘要）
  - Introduction（引言）
  - Related Work（相关工作）
  - Method（方法）
  - Experiment（实验）
  - Conclusion（结论）
- 每个章节的生成以对应的课题模块文档作为输入上下文，并支持一键汇总为完整 Manuscript
- 用户可对生成内容进行编辑、重新生成、分段调整，并锁定已确认章节或段落
- 提供章节间的一致性检查，覆盖术语、研究贡献、数据集名称、指标、引用关系等关键信息
- 支持在整篇草稿视图中统一查看章节衔接、行文重复与引用完整性问题

**验收标准：**

- 各章节可独立触发生成，互不干扰，也可汇总为完整 Manuscript
- 已锁定内容在重新生成时不被覆盖
- 生成内容与课题文档中的事实信息保持一致

### 4.10 会议追踪

**需求描述：**  
集中管理科研人员关注的学术会议的关键时间节点和投稿要求。

**详细需求：**

- 支持追踪多个学术会议，包括但不限于 NeurIPS、AAAI、CVPR、ICCV、ECCV、ACL、ICLR 等
- 每个会议的追踪信息包含：
  - 论文投稿截止日期（摘要截止日期、全文截止日期、截止时区）
  - 审稿结果通知时间
  - Camera-ready 截止时间
  - 特殊投稿要求（匿名规则、页数限制、附件要求等）
- 信息**定期自动刷新**，保持最新状态
- 支持用户手动添加或修改会议信息
- 重要时间节点到期前发出提醒

**验收标准：**

- 会议信息定期自动更新，更新频率用户可配置
- 时间节点提醒在截止日前指定天数内触发

## 5. 知识图谱需求

**需求描述：**  
以可视化图谱的形式呈现论文、数据集、课题、观点之间的关联关系，支持全局知识浏览与局部探索。

### 5.1 图谱类型

| 图谱名称 | 节点类型 | 关系说明 |
|----------|---------|---------|
| **数据集 ↔ 论文** | 数据集、论文 | 论文使用了某数据集；数据集被哪些论文使用 |
| **论文 ↔ 课题** | 论文、课题 | 论文作为某课题的相关工作；论文启发了某课题的方法；等多种关联类型 |
| **观点 ↔ 论文** | 观点、论文 | 观点来源于某论文；某论文支持/反驳了某观点 |

### 5.2 可视化需求

- 以图形界面展示图谱，节点和边可交互点击查看详情
- 自动抽取的关系默认只读；用户可对手动补充的节点说明与关系标注进行编辑
- 点击节点可查看对应论文/数据集/课题的详情摘要
- 支持按类型筛选展示部分图谱
- 支持搜索定位特定节点
- 图谱随数据变更自动更新

**验收标准：**

- 各类图谱均可独立浏览
- 节点点击后可跳转到对应详情页面
