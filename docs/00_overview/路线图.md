# Research-Flow 路线图

## 当前阶段：Paper 全流程

目标是在不引入数据库、不引入 LLM Provider、不展开平台后续模块的前提下，完成一个可维护的 Paper 工作流软件。

## M0：范围冻结

- 根 `data/` 作为唯一真源。
- JSON 暂存状态，暂不引入数据库。
- 人工完成分类、保留、拒绝、修正和质量判断。
- 只实现 Paper 全流程，不实现 Project / Dataset / Knowledge。

## M1：数据与状态层

- 固化 `data/Discover`、`data/Acquire`、`data/Library`、`data/templates`。
- 补齐 batch、queue、paper、parser run 的 JSON 状态文件。
- 保证 `scan` 可重建首页和列表视图所需的索引。
- 明确冲突、失败、待人工处理的状态写入规则。

## M2：Paper 程序式处理链路

- 完成 `ingest / migrate / note-template`。
- 接入真实 PDF parser。
- 保存解析文本、结构、图片索引和运行日志。
- 失败时写入 `failed` 或 `needs-review`，不能静默跳过。

## M3：前端页面闭环

- HomePage：全局状态入口。
- Discover：搜索批次和候选论文查看。
- Acquire：待获取、待补 PDF、待人工确认队列。
- Library：正式文献库列表、筛选和状态统计。
- Paper Detail：单篇论文元数据、PDF、解析产物和 note。
- Config：当前数据根、路径检查、parser 配置和后续数据目录扩展入口。

## M4：接口与测试收口

- 补齐页面需要的 API。
- 补齐 CLI 与 API 的共享服务层测试。
- 补齐真实 PDF 样例解析测试。
- 固化最小回归命令。

## 后续阶段：从 master 大方案提炼

- 可配置外部数据根目录。
- 数据库持久化与迁移。
- Project / Dataset / Knowledge 模块。
- LLM Provider、Skill 手动触发和后续 Agent 编排。
- 推荐、Daily Feed、会议追踪和更多平台页面。
