# 文献管理

页面整体延续左右分栏布局：左侧为系统全局导航与当前视图结构树，右侧为主内容区。本文重点定义 `Library` 页面特有的“结构树 + 卡片式列表”布局，以及相应的信息层级与核心交互方式。

## 1. 顶部全局控制区

- 页面主标题为 `Paper Library`，采用高对比度粗体，强化学术文献管理场景的专业感。
- 标题右侧放置两个核心操作按钮：
  - `Filter`：简洁的浅灰色按钮，带筛选图标，用于展开高级筛选面板，例如按 `CCF Rank`、年份或阅读状态筛选。
  - `Import PDF`：高对比度深色按钮，带加号图标，作为文献手动入库的主入口。

## 2. 左侧归档结构树

- 该区域位于全局导航右侧，作为 `Library` 专属的二级导航，承担类似 Zotero 文件夹系统的目录组织作用。
- 支持多级目录嵌套：
  - 父级目录例如 `Machine Learning`，应带有展开 / 折叠箭头。
  - 子级目录例如 `Generative Models`，通过缩进体现层级关系。
- 当前选中目录使用深色加粗字体，并辅以左侧细竖线或极轻背景高亮，作为 `Active State`。
- 未选中目录保持浅灰色，以降低视觉干扰。

## 3. 列表视图表头

- 列表顶部采用“隐形表头”设计，不使用传统表格边框，而是通过与下方数据严格对齐的字段标签建立阅读秩序。
- 表头字段包括：
  - `Authors`
  - `Conformance / Venue`
  - `CCF Rank`
  - `IF`
  - `Year`
  - `Status`
- 表头字体统一为小字号、浅灰色无衬线样式，并适当拉开字距，主要承担信息提示与对齐参考作用，不喧宾夺主。

## 4. 文献卡片列表区

内容区采用垂直排列的宽体卡片列表。卡片整体为纯白底色、大圆角与柔和弥散阴影，卡片之间保持适中的纵向间距，以兼顾信息密度与浏览舒适度。每张论文卡片由上下两层结构组成。

### 4.1 标题与快捷操作层

- 论文标题使用优雅的衬线字体，字号适中且加粗，作为卡片的第一视觉焦点。
- 标题下方补充一行极浅灰色的斜体副标题或一句话摘要，用于提升信息密度，并帮助用户快速回忆论文内容。
- 标题行右侧固定展示三个快捷操作入口：
  - `PDF`：微型红色圆角按钮，点击后在侧边栏或新标签页中预览原始论文。
  - `Notes`：微型蓝色圆角按钮，点击后跳转至该论文对应的 `LLM.md` 或 `HUMAN.md` 分析页面。

### 4.2 元数据展示层

- 该层字段需与顶部表头严格垂直对齐，保证扫读效率。
- `Authors`：截断显示作者信息，例如 `Vaswani, A., et al.`，并使用浅灰色文字。
- `Conformance / Venue`：采用斜体展示会议或期刊名称，统一使用缩写，例如 `CVPR`、`AAAI`，并使用浅灰色文字。
- `CCF Rank`：使用小型圆角徽章展示评级，例如浅紫蓝底色搭配深色字母 `A` 或 `B`。
- `IF` 与 `Year`：采用纯文本展示；若无数据，则统一显示 `N/A`。
- `Status`：位于最右侧，由彩色圆点与全大写状态文本共同组成，例如：
  - `🟢 Annotated`：已批注或已分析。
  - `🟠 In Process`：处理中或阅读中。
  - `⚪ UNREAD`：尚未阅读。

## 5. 底部分页组件

- `Library` 页面使用传统分页器，而非 `Daily` 页面的滚动加载。
- 该设计更符合长期文献库场景下“精确定位”和“归档检索”的使用需求。
- 分页组件居中放置在列表底部，包含：
  - 上一页按钮 `<`
  - 下一页按钮 `>`
  - 当前页高亮的数字页码按钮，例如黑底白字的 `1`

## 6. 与 `Library.png` 的差异

1. `Filter` 和 `Import PDF` 不使用胶囊按钮，而改为更简洁的扁平化设计，突出功能性而非装饰性。
2. 字体大小控制在适中范围，避免字号过大导致信息拥挤，同时保留足够的视觉层次。
3. 取消论文右侧的 `More` 按钮，改为右键操作菜单。
4. 增加 `Note` 按钮，作为两个 markdown 分析页面的直接入口，提升访问效率。
5. 当 `PDF` 或 `Note` 缺失时，对应按钮降低透明度但仍保持可点击；点击后弹出提示，引导用户补全文档，而不是直接隐藏按钮，以保留功能可发现性。
