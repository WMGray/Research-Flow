### 11.1 测试分层

| 层级 | 测试内容 | 建议覆盖 |

## 12. 风险登记册

| 风险 | 影响 | 触发信号 | 缓解策略 | 优先级 |
|------|------|----------|----------|--------|
| Windows pytest 临时目录 ACL | 无法判断后端测试真实状态 | `PermissionError`、`pytest-of-*` 或 `.uv-cache` 不可读 | 固定仓库内可写 basetemp；清理坏 ACL；增加 import smoke | P0 |
| 缺失 `knowledge.runtime` | import 链路不稳定，Worker/API 启动可能失败 | `ModuleNotFoundError` | 先补最小 runtime，再迭代真实抽取 | P0 |
| SQLite 并发写入 | Celery 异步化后 Job/Paper 状态可能锁表 | `database is locked`、任务随机失败 | 短事务、重试、单 writer 约束；必要时迁移 Postgres | P0 |
| API envelope 漂移 | 前端错误解析和契约测试不稳定 | 文档 `{code,message,data}` 与实现 `{data,meta,error}` 不一致 | 先定最终 envelope，再统一 docs/API client/tests | P1 |
| Jobs 状态语义不统一 | 通知中心和 pipeline 进度展示混乱 | 同一任务无法区分 queued/running/stage | 统一 JobRecord 字段和状态机，必要时增加 `stage` | P1 |
| LLM/provider 失败 | Knowledge/Dataset/Project 生成不可用 | timeout、quota、JSON parse failure | gated real-provider tests；fallback message；JSON repair + schema validation | P1 |
| 前端静态页与后端能力漂移 | 看起来有页面但没有真实数据闭环 | 页面仍使用 mock data | 接口覆盖矩阵逐项打勾，禁止只做静态 UI 验收 | P1 |
| Celery/Redis 环境缺失 | 异步任务本地无法启动 | Worker 启动失败、broker 连接失败 | 支持 eager mode 测试；文档化本地启动命令；健康检查 | P1 |
| 大文件 `PaperRepository` 继续膨胀 | 后续修改冲突和回归风险上升 | 单文件超过 2500 行，职责混杂 | 按 documents/artifacts/pipeline/note_state 渐进拆分 | P1 |
| 推送粒度失控 | 梭哈阶段难以回滚 | 一个 commit 混多个目标 | 强制小 goal commit，清单回填 commit hash | P1 |
