# Research-Flow 表设计文档

## 1. 设计目标

后端表设计需要同时满足以下要求：

- 支撑本地文件与数据库索引协同管理
- 支撑 `Paper`、`Project`、`Presentation`、`Dataset`、`Knowledge` 五类核心资源
- 支撑 `note.md` 单笔记策略
- 支撑“PDF 解析 -> LLM 优化解析 -> 章节划分 -> note 生成”的处理链路
- 支撑异步任务、模型配置与 Prompt 模板管理
- 为 `P2` 的 `Zotero`、每日推荐、会议追踪预留扩展空间

---

## 2. 分层原则

本系统采用“物理层 - 资产层 - 关系层 - 业务层”的四段式设计。

### 2.1 物理层

物理层只关心文件真实位置与存储引擎。

### 2.2 资产层

资产层为所有文件、目录与逻辑对象提供统一的全局标识。

### 2.3 关系层

关系层记录资源之间的显式关联，承担图结构与引用网络的基础功能。

### 2.4 业务层

业务层保存各类科研资源的领域字段，并通过 `asset_id` 挂接到资产层。

## 3. 基础表

### 3.1 物理存根表 `physical_item`

| 字段名 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `item_id` | `UUID` | 主键 | 物理实体唯一 ID |
| `storage_path` | `VARCHAR(512)` | 非空 | 文件系统路径或对象存储 URI |
| `storage_type` | `VARCHAR(20)` | 非空 | `LOCAL` / `OSS` / `S3` / `MINIO` |
| `mime_type` | `VARCHAR(100)` | - | MIME 类型 |
| `ref_count` | `INT` | 默认 `0` | 被资产层引用次数 |
| `created_at` | `TIMESTAMP` | 非空 | 创建时间 |

### 3.2 资产注册表 `asset_registry`

| 字段名 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `asset_id` | `BIGINT` | 主键，自增 | 全局资源逻辑 ID |
| `item_id` | `UUID` | 外键，可空 | 指向 `physical_item.item_id` |
| `display_name` | `VARCHAR(255)` | 非空 | 显示名 |
| `asset_type` | `VARCHAR(30)` | 索引 | `Paper` / `Project` / `Presentation` / `Dataset` / `Knowledge` / `Markdown` / `Pdf` / `Dir` / `Attachment` |
| `file_ext` | `VARCHAR(20)` | - | 扩展名 |
| `media_type` | `VARCHAR(100)` | - | 资源媒体类型 |
| `file_size` | `BIGINT` | - | 文件大小 |
| `content_hash` | `VARCHAR(64)` | 唯一索引，可空 | 内容指纹 |
| `logical_path` | `VARCHAR(512)` | 索引，可空 | 平台内部逻辑路径 |
| `is_deleted` | `BOOLEAN` | 默认 `FALSE` | 逻辑删除标记 |
| `created_at` | `TIMESTAMP` | 非空 | 创建时间 |
| `updated_at` | `TIMESTAMP` | 非空 | 更新时间 |

### 3.3 资产关系表 `asset_link`

| 字段名 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `link_id` | `BIGINT` | 主键，自增 | 关系记录 ID |
| `source_id` | `BIGINT` | 索引 | 起始资源 ID |
| `target_id` | `BIGINT` | 索引 | 目标资源 ID |
| `relation_type` | `VARCHAR(50)` | 索引 | 关系类型 |
| `relation_meta` | `JSON` | 可空 | 关系元数据 |
| `sort_order` | `INT` | 默认 `0` | 排序 |
| `created_at` | `TIMESTAMP` | 非空 | 创建时间 |

**推荐关系类型**

- `CONTAINS`
- `REFERENCES`
- `USES`
- `EXTRACTED_FROM`
- `DERIVED_FROM`
- `GENERATED_BY_JOB`
- `BELONGS_TO_CATEGORY`

## 6. `P0` 最小落地表集合

首轮后端实现建议最少落地以下表：

- `physical_item`
- `asset_registry`
- `asset_link`
- `biz_paper`
- `biz_doc_layout`
- `biz_category`
- `biz_job`
- `sys_agent_profile`
- `sys_prompt_template`
- `sys_llm_probe_result`

在此基础上可完成：

- 论文入库
- PDF 下载
- PDF 解析
- LLM 优化解析
- `note.md` 生成与编辑
- 分类管理
- 任务跟踪
- `LLM / Agent` 配置管理
