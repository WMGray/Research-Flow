# 数据模型

## 1. 文档目标

本文档用于定义 Research-Flow 后端的数据模型与对象边界，作为数据库设计、文件组织、异步任务编排与 API 设计的统一依据。

当前版本仅覆盖后端资源模型与服务能力，不展开前端页面与交互需求。

---

## 2. 建模原则

### 2.1 资源中心化

系统围绕五类核心资源组织后端能力：

- `Paper`
- `Project`
- `Presentation`
- `Dataset`
- `Knowledge`

下载、解析、LLM 优化解析、笔记生成、知识提炼、PPT 草稿生成等能力均作为资源上的操作存在，不作为独立产品主线。

### 2.2 本地优先

- 论文 PDF、研究笔记、课题文档、汇报草稿等核心材料优先落地到本地文件系统。
- 数据库存储索引、状态、关系与任务信息。
- 外部服务仅提供解析、模型调用、同步与候选数据接入能力。

### 2.3 文件系统与数据库分工

- 文件系统负责保存正文内容、二进制文件与阶段产物。
- 数据库负责保存资源主键、元数据、资源关系、任务状态、配置与检索索引。
- 任一资源都应能在数据库中定位，在文件系统中追溯。
- `SQLite` 的组织思路与表设计保持一致，采用“物理层 - 资产层 - 关系层 - 业务层”的四段式设计。

### 2.4 单一研究笔记策略

- 论文级分析文档统一为 `note.md`。
- 系统生成的分析结果直接写入 `note.md`，用户在同一文档中继续编辑。
- 后端需要提供版本保护与重生成策略，避免异步任务覆盖人工修改内容。

### 2.5 关系显式化

- 论文与课题、论文与数据集、论文与 Knowledge、课题与汇报之间的关系必须显式建模。
- 关系模型需支持来源记录、关系类型、时间戳与扩展元数据。

### 2.6 异步任务化

- 下载、PDF 解析、LLM 优化解析、笔记生成、知识提炼、PPT 草稿生成统一抽象为 `Job`。
- 同步接口只负责创建任务、读取状态、必要时取消任务。

### 2.7 配置独立化

- `LLM` / `Agent` 配置、Prompt 模板与特性开关为独立系统对象。
- 业务资源不直接持有硬编码模型配置。
- `Agno` 作为统一的 LLM / Agent 管理框架，负责 scene / skill 到 provider、prompt、tool 的绑定。

## 3. 核心资源模型

| 资源 | 说明 | 核心内容 | 主要关系 | 优先级 |
|------|------|----------|----------|--------|
| `Paper` | 已纳入平台管理的论文实体 | 元数据、`paper.pdf`、`note.md`、解析产物 | 关联 `Project`、`Dataset`、`Knowledge` | `P0` |
| `Project` | 课题与研究项目实体 | `overview.md`、`related-work.md`、`method.md`、`experiment.md`、`conclusion.md`、`manuscript.md` | 关联 `Paper`、`Presentation`、`Dataset` | `P1` |
| `Presentation` | 组会汇报或阶段汇报实体 | `outline.md`、`slides.md`、`speaker-notes.md`、导出文件 | 从 `Project` 衍生，引用 `Paper` 与 `Knowledge` | `P1` |
| `Dataset` | 数据集档案与实验对比实体 | 数据集描述、访问链接、结果聚合 | 被 `Paper` 和 `Project` 使用 | `P1` |
| `Knowledge` | 从论文中提炼的结构化知识实体 | 观点、定义、特征、证据片段 | 来源于 `Paper`，服务于 `Project` 与 `Presentation` | `P1` |

## 4. 支撑对象模型

| 对象 | 说明 | 优先级 |
|------|------|--------|
| `Category` | 论文分类树，支持方向与子方向归类 | `P0` |
| `Job` | 异步任务实体，统一跟踪下载、解析、生成等操作 | `P0` |
| `AgentProfile` | 模型与能力配置项，定义 provider、model、feature bindings | `P0` |
| `PromptTemplate` | Prompt 模板管理对象，支持版本化与启停 | `P0` |
| `SkillBinding` | 基于 `Agno` 的 skill 场景绑定，定义 skill、scene、agent profile、prompt 与工具集的映射 | `P0` |
| `LLMProbeResult` | 已配置 LLM 的连通状态、首字时延与最近探测结果 | `P0` |
| `FeedItem` | 每日推荐与候选论文池条目，尚未入正式论文库 | `P2` |
| `ZoteroBinding` | 本地资源与 Zotero 条目之间的映射关系 | `P2` |
| `Conference` | 截止时间、通知时间等会议信息 | `P2` |

## 5. 资源定义

### 5.1 Paper

`Paper` 是系统最核心的研究资产，承担论文入库、下载、解析、LLM 优化解析、章节划分、笔记生成与后续知识提炼的起点角色。

**核心字段**

- `paper_id`
- `paper_slug`
- `paper_stage`
- `title`
- `authors`
- `pub_year`
- `venue`
- `doi`
- `source_url`
- `pdf_url`
- `category_id`
- `download_status`
- `parse_status`
- `refine_status`
- `review_status`
- `note_status`

**核心文件**

- `paper.pdf`
- `note.md`
- `parsed/raw.md`
- `parsed/refined.md`
- `parsed/sections/*.md`

**处理链路**

- `metadata_ready`：获得最小元信息后即完成入库，可供 `Project` 关联
- `download`：获取并保存 `paper.pdf`
- `parse`：生成解析器原始输出 `parsed/raw.md`
- `refine`：使用 LLM 对解析结果进行清理与结构优化，生成 `parsed/refined.md`
- `review`：用户直接在 `parsed/refined.md` 上进行人工审查与确认
- `split`：基于优化后的文档进行章节切分，生成四类 canonical sections
- `note`：基于章节内容生成 `note.md`

**主要关系**

- `Paper -> Project`：相关工作、方法参考、baseline、灵感来源等
- `Paper -> Dataset`：使用、评测、对比
- `Paper -> Knowledge`：提炼来源

### 5.2 Project

`Project` 用于管理课题目标、阶段产出与长期写作上下文。

**核心字段**

- `project_id`
- `project_slug`
- `name`
- `status`
- `summary`
- `owner`

**核心文档**

- `overview.md`
- `related-work.md`
- `method.md`
- `experiment.md`
- `conclusion.md`
- `manuscript.md`

**主要关系**

- `Project -> Paper`
- `Project -> Presentation`
- `Project -> Dataset`

### 5.3 Presentation

`Presentation` 用于管理组会汇报、阶段汇报或答辩材料。

**核心字段**

- `presentation_id`
- `project_id`
- `title`
- `status`
- `scene_type`
- `export_format`

**核心文档**

- `outline.md`
- `slides.md`
- `speaker-notes.md`

**主要关系**

- `Presentation -> Project`
- `Presentation -> Paper`
- `Presentation -> Knowledge`

### 5.4 Dataset

`Dataset` 是聚合多个论文的数据集实体，不是单篇论文中的一个简单字段。

它用于：

- 汇总使用同一数据集的论文
- 沉淀数据集描述与 benchmark 结果
- 为 `Project` 与横向比较提供聚合视角

**核心字段**

- `dataset_id`
- `name`
- `normalized_name`
- `aliases`
- `task_type`
- `description`
- `access_url`

**主要关系**

- `Dataset -> Paper`
- `Dataset -> Project`

说明：

- 单篇论文先抽出 dataset mention
- 后续再归一化到聚合的 `Dataset` 实体

### 5.5 Knowledge

`Knowledge` 是结构化知识资源，不等同于原始论文笔记。

**知识类型**

- `view`
- `definition`
- `feature`

**核心字段**

- `knowledge_id`
- `knowledge_type`
- `title`
- `summary_zh`
- `original_text_en`
- `category_label`
- `research_field`
- `source_paper_id`
- `source_section`
- `source_locator`
- `evidence_text`
- `confidence_score`
- `review_status`
- `llm_run_id`

说明：

- `summary_zh` 用于承接中文总结
- `original_text_en` 用于保留英文原文
- `knowledge_type` 用于区分定义 / 观点 / 特征
- `category_label` 用于记录所属类别、主题或研究方向
- `source_section` 与 `source_locator` 用于追溯来源位置
- `review_status` 用于表示是否经过人工确认或修订
- `confidence_score >= 0.6` 时允许先入库
- 初始 `review_status` 建议为 `pending_review`

**主要关系**

- `Knowledge <- Paper`
- `Knowledge -> Project`
- `Knowledge -> Presentation`

### 5.6 AgentProfile / PromptTemplate

该类对象属于系统配置域，不直接面向学术资源，但直接决定下载、解析、生成等任务的执行策略。

**AgentProfile 核心字段**

- `profile_key`
- `provider`
- `model_name`
- `scene`
- `temperature`
- `enabled`

**PromptTemplate 核心字段**

- `template_key`
- `scene`
- `version`
- `content`
- `enabled`

### 5.7 SkillBinding

`SkillBinding` 用于表达“某个能力场景如何以 skill 的方式由 `Agno + LLM` 执行”。

适用场景包括：

- Knowledge 抽取
- 章节切分
- note 生成
- related work 整理
- method 草稿生成
- experiment 建议

**核心字段**

- `skill_key`
- `scene`
- `agent_profile_key`
- `prompt_template_key`
- `toolset`
- `enabled`

### 5.8 LLMProbeResult

`LLMProbeResult` 用于描述已配置 `LLM` 的运行可用性，不作为学术资源持久化对象，而作为系统观测与运维支撑对象存在。

**核心字段**

- `profile_key`
- `provider`
- `model_name`
- `connectivity_status`
- `ttft_ms`
- `checked_at`
- `error_message`

### 5.9 Job

`Job` 用于统一管理长耗时操作。

**任务类型**

- `paper_download`
- `paper_parse`
- `paper_refine_parse`
- `paper_split_sections`
- `paper_generate_note`
- `paper_extract_knowledge`
- `paper_extract_datasets`
- `presentation_generate_outline`
- `presentation_generate_slides`
- `llm_connectivity_probe`

**状态**

- `queued`
- `running`
- `waiting_review`
- `waiting_confirm`
- `succeeded`
- `failed`
- `cancelled`

## 6. 关系模型

后端关系模型至少需要支持以下语义：

- `CONTAINS`：父资源包含子资源或文档
- `REFERENCES`：课题或汇报引用论文
- `USES`：论文或课题使用数据集
- `EXTRACTED_FROM`：Knowledge 来源于论文
- `DERIVED_FROM`：汇报或导出文件来源于课题文档或论文材料
- `GENERATED_BY_JOB`：某个资源由异步任务产出
- `BELONGS_TO_CATEGORY`：论文归属分类

关系记录应支持附加元数据，例如关系来源、引用角色、排序顺序和创建时间。

## 7. 文件系统映射

### 7.1 顶层目录

- `Databases/`
- `Papers/`
- `Projects/`
- `Datasets/`
- `Archives/`

### 7.2 Paper 映射

```text
Papers/{category_path}/{paper_slug}/
  paper.pdf
  note.md
  parsed/
    raw.md
    refined.md
    sections/
      01-introduction.md
      02-method.md
```

说明：

- 目录统一采用 slug 化路径
- 人工审查直接复用 `refined.md`，不创建额外的 `review.md`

### 7.3 Project 映射

```text
Projects/{project_slug}/
  overview.md
  related-work.md
  method.md
  experiment.md
  conclusion.md
  manuscript.md
  presentations/
    {presentation_slug}/
      outline.md
      slides.md
      speaker-notes.md
      exports/
```

### 7.4 Dataset 映射

```text
Datasets/{dataset_slug}/
  dataset.md
  benchmarks.md
```

### 7.5 Knowledge 映射

`Knowledge` 以数据库存储为主，必要时支持导出快照；不强制要求独立 Markdown 文件。

## 8. 状态模型

### 8.1 Paper

`Paper` 的主串行状态建议统一收敛到 `paper_stage`：

- `metadata_ready`
- `downloaded`
- `parsed`
- `refined`
- `review_confirmed`
- `sectioned`
- `noted`
- `knowledge_extracted`
- `dataset_extracted`
- `completed`
- `error`

说明：

- `paper_stage` 用于表达论文链路推进到了哪一阶段
- 在 `metadata_ready` 阶段即允许被 `Project` 关联

### 8.2 Project

- `active`
- `paused`
- `archived`

### 8.3 Presentation

- `draft`
- `generating`
- `ready`
- `exported`
- `archived`

### 8.4 Job

- `queued`
- `running`
- `waiting_review`
- `waiting_confirm`
- `succeeded`
- `failed`
- `cancelled`

### 8.5 note_status

为保证 `note.md` 的状态可被准确识别，建议单独维护 `note_status`：

- `clean_generated`
- `user_modified`
- `merged`
- `conflict_pending`

识别信号至少包括：

- 最近一次系统生成时间
- 最近一次用户修改时间
- 文档整体 hash
- 系统托管区块 hash
- 最近一次合并结果

### 8.6 Knowledge review_status

建议至少支持：

- `pending_review`
- `approved`
- `rejected`
- `edited`

## 9. 后端优先级

### `P0`

- `Paper` 入库、下载、解析、LLM 优化解析、`note.md` 生成与编辑
- `Category` 管理与检索
- `Job` 队列与状态查询
- `AgentProfile` / `PromptTemplate` / 模型绑定配置
- `LLMProbeResult` 查询与连通性探测

### `P1`

- `Project`
- `Presentation`
- `Dataset`
- `Knowledge`

### `P2`

- `FeedItem`
- `ZoteroBinding`
- `Conference`
- 知识图谱复杂可视化支撑接口
