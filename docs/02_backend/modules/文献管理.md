# 文献管理模块

## 1. 模块目标

把已有论文资料整理流程沉淀为可重复执行的本地 Paper 全流程。

## 2. 目录流转

```text
Discover/search_batches
  -> Acquire/curated
  -> Acquire/needs-pdf
  -> Acquire/needs-review
  -> Library/<Domain>/<Area>/<Topic>/<slug>
  -> Library/unclassified/<slug>
  -> parsed outputs
```

## 3. 当前能力

- 扫描批次
- 扫描论文目录
- 读取当前兼容的 `metadata.yaml`
- 目标状态写入 `metadata.json`
- 读取 `note.md` front matter
- 合并 metadata
- ingest
- migrate
- note 模板生成

## 4. 目标能力

- Discover 批次扫描和候选论文状态管理
- Acquire 队列人工确认
- PDF 存在性检查
- Library 入库与目录规范化
- 程序式 PDF parser
- parser 产物索引
- Paper Detail 所需单篇聚合数据
- Config 所需路径健康检查

## 5. 人工节点

当前不引入 LLM Provider。以下节点先由人工处理：

- 候选论文 keep / reject
- 分类目录确认
- 缺失 PDF 补充
- 解析失败后的重跑或跳过
- note 内容质量判断
- 冲突目录合并

## 6. 冲突处理

如果目标目录已经存在：

- 不覆盖已有手工内容
- 将目标论文状态标记为 `needs-review`
- 在 `metadata.json` 中写入 `Target already exists`

## 7. 后续扩展方向

- 接入 refined / images 产物链路
- 接入手动 skill 调用
- 支持外部数据根目录
- 接入数据库和任务队列
