# Paper 全流程接口规范

## 1. 基础约定

- Base URL: `http://127.0.0.1:8000`
- 前缀：`/api`
- 响应格式：

```json
{
  "ok": true,
  "data": {},
  "error": null
}
```

## 2. 当前已实现接口

### Dashboard

- `GET /api/dashboard/home`
- `GET /api/dashboard/papers-overview`
- `GET /api/dashboard/discover`
- `GET /api/dashboard/acquire`
- `GET /api/dashboard/library`

### Papers

- `GET /api/papers`
- `GET /api/papers/{paper_id}`
- `POST /api/papers/ingest`
- `POST /api/papers/migrate`
- `POST /api/papers/generate-note`
- `POST /api/papers/{paper_id}/generate-note`
- `POST /api/papers/{paper_id}/parse-pdf`
- `GET /api/papers/{paper_id}/parser-runs`
- `POST /api/papers/{paper_id}/accept`
- `POST /api/papers/{paper_id}/mark-review`
- `POST /api/papers/{paper_id}/mark-processed`
- `POST /api/papers/{paper_id}/reject`

### Workflow 正式动作

- `parse-pdf`
- `accept`
- `generate-note`
- `delete`（API 名称仍为 `reject`，语义为物理删除）

### 兼容/内部工具接口

- `POST /api/papers/ingest`
- `POST /api/papers/migrate`
- `POST /api/papers/{paper_id}/mark-review`
- `POST /api/papers/{paper_id}/mark-processed`

### Discover

- `POST /api/discover/batches/{batch_id}/candidates/{candidate_id}/decision`

### Config

- `GET /api/config`

## 3. Paper 返回契约

`GET /api/papers`、`GET /api/papers/{paper_id}`、动作类接口都返回 `PaperRecord`。

### 新契约字段

- `stage`: `discover | acquire | library`
- `asset_status`: `missing_pdf | pdf_ready`
- `parser_status`: `not_started | running | parsed | failed`
- `review_status`: `pending | accepted`
- `note_status`: `missing | template | ready`
- `parser_artifacts`:
  - `text_path`
  - `sections_path`
  - `refined_path`
- `capabilities`:
  - `parse`
  - `accept`
  - `generate_note`
  - `delete`

### 兼容字段

- `status` 仍保留一轮兼容
- `mark-review`、`mark-processed` 仍保留一轮兼容
- 新前端不得再用旧 `status` 作为主流程真源

## 4. 主流程语义

### Discover -> Acquire

- 候选论文经 `keep` 后进入 `Acquire/curated/...`

### Acquire -> Library

- 使用 `POST /api/papers/{paper_id}/accept`
- 仅当 `parser_status == parsed` 时允许
- 会把目录从 `Acquire/curated/...` 真正迁移到 `Library/...`

### Delete

- 使用 `POST /api/papers/{paper_id}/reject`
- 继续执行物理删除，不做归档

## 5. 兼容状态投影

旧 `status` 现在是从新状态投影得到：

- `needs-pdf`
- `parse-pending`
- `parse-failed`
- `needs-review`
- `processed`

## 6. 当前边界

- 暂无数据库分页查询
- 暂无异步任务系统
- 暂无 LLM Provider 配置 API
- 暂无 Project / Dataset / Knowledge API
