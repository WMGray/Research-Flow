## 1. 文档范围

本文档定义 Research-Flow 后端 API 的接口约定，覆盖两类内容：

- **已实现接口**：当前 FastAPI 服务已经暴露，可直接调用。
- **规划接口**：根据 URD/README/P0-P1 需求整理出的目标契约，供后续实现、测试和前端联调用。

当前后端版本为 `0.1.0`。已实现接口以代码为准，规划接口在实现前可根据数据模型继续调整。

## 2. 状态标记

| 状态 | 含义 |
|------|------|
| `已实现` | 当前代码已暴露该接口 |
| `规划-P0` | 核心闭环接口，优先实现 |
| `规划-P1` | 重要增强接口，P0 稳定后实现 |
| `规划-P2` | 增强能力或低频能力 |

## 3. 通用约定

### 3.1 Base URL

| 环境 | Base URL |
|------|----------|
| 本地开发 | `http://localhost:8000` |
| Swagger | `http://localhost:8000/docs` |
| OpenAPI JSON | `http://localhost:8000/openapi.json` |

### 3.2 路由版本

当前已实现接口暂未统一使用 `/api/v1` 前缀：

- `/health`
- `/services/mcp`
- `/services/mcp/tools`
- `/paper-download/*`

后续业务接口统一使用：

```text
/api/v1/{resource}
```

现有 `/paper-download/*` 可作为兼容接口保留，后续建议在 `/api/v1/papers/*` 下提供业务化封装。

### 3.3 请求格式

- 默认请求体为 `application/json`。
- 文件上传使用 `multipart/form-data`。
- 时间字段使用 ISO 8601 字符串，例如 `2026-04-23T09:00:00+08:00`。
- ID 字段统一使用 `integer`，对应后端资产层 `asset_id` 或业务表主键。
- 长耗时任务默认异步执行，返回 `202 Accepted` 与 `job_id`。

### 3.4 响应格式

已实现接口当前直接返回 Pydantic response model。

新增 `/api/v1` 业务接口建议统一使用 envelope：

```json
{
  "data": {},
  "meta": {},
  "error": null
}
```

错误响应建议统一为：

```json
{
  "data": null,
  "meta": {},
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Human-readable error message.",
    "details": {}
  }
}
```

### 3.5 常用 HTTP 状态码

| 状态码 | 含义 |
|--------|------|
| `200` | 请求成功 |
| `201` | 资源创建成功 |
| `202` | 异步任务已接受 |
| `204` | 删除或空响应成功 |
| `400` | 请求语义错误 |
| `401` | 未认证，预留 |
| `403` | 无权限，预留 |
| `404` | 资源不存在 |
| `409` | 资源冲突，例如重复入库 |
| `422` | 参数校验失败 |
| `500` | 服务内部错误 |
| `502` | 外部服务失败，例如 MinerU / Zotero MCP |
| `504` | 外部服务超时 |

### 3.6 分页与排序

列表接口统一支持：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `page` | integer | `1` | 页码，从 1 开始 |
| `page_size` | integer | `20` | 每页数量，建议最大 100 |
| `sort` | string | `updated_at` | 排序字段 |
| `order` | string | `desc` | `asc` 或 `desc` |

分页响应 `meta`：

```json
{
  "page": 1,
  "page_size": 20,
  "total": 128
}
```

### 4.4 论文解析下载：Resolve

| 项目 | 说明 |
|------|------|
| 状态 | `已实现` |
| Method | `POST` |
| Path | `/paper-download/resolve` |
| 说明 | 根据 URL / DOI / 论文名解析论文元数据与可下载 PDF URL，但不下载文件 |

#### Request Body

`url`、`doi`、`name` 三者必须且只能提供一个。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `url` | string/null | 否 | 论文 URL |
| `doi` | string/null | 否 | DOI |
| `name` | string/null | 否 | 论文名称或搜索关键词 |
| `title` | string | 否 | 已知标题，用于辅助解析 |
| `year` | string | 否 | 已知年份，用于辅助解析 |
| `venue` | string | 否 | 已知会议/期刊，用于辅助解析 |

#### 请求示例

```json
{
  "doi": "10.48550/arXiv.2106.09685"
}
```

#### Response Body

| 字段 | 类型 | 说明 |
|------|------|------|
| `raw_input` | string | 原始输入 |
| `title` | string | 解析到的论文标题 |
| `year` | string | 年份 |
| `venue` | string | 来源会议/期刊 |
| `doi` | string | DOI |
| `resolve_method` | string | 解析方法 |
| `source` | string | 元数据或 PDF 来源 |
| `status` | string | 解析状态，例如 `ready_download` |
| `pdf_url` | string | PDF URL |
| `landing_url` | string | 论文落地页 |
| `final_url` | string | 最终 URL |
| `http_status` | string | 探测 HTTP 状态 |
| `content_type` | string | 探测内容类型 |
| `detail` | string | 细节说明 |
| `error_code` | string | 错误码，成功时为空 |
| `metadata_source` | string | 元数据来源 |
| `metadata_confidence` | string | 元数据置信度 |
| `suggested_filename` | string | 建议文件名 |
| `target_path` | string | 目标保存路径 |
| `probe_trace` | array[string] | 解析探测轨迹 |

#### 响应示例

```json
{
  "raw_input": "10.48550/arXiv.2106.09685",
  "title": "LoRA: Low-Rank Adaptation of Large Language Models",
  "year": "2021",

  "resolve_method": "direct",
  "source": "arxiv",
  "status": "ready_download",
  "pdf_url": "https://arxiv.org/pdf/2106.09685",
  "landing_url": "https://arxiv.org/abs/2106.09685",
  "final_url": "https://arxiv.org/pdf/2106.09685",
  "http_status": "200",
  "content_type": "application/pdf",
  "detail": "",
  "error_code": "",
  "metadata_source": "arxiv",
  "metadata_confidence": "high",
  "suggested_filename": "LoRA Low-Rank Adaptation of Large Language Models__2021.pdf",
```

### 4.5 论文解析下载：Download

| 项目 | 说明 |
|------|------|
| 状态 | `已实现` |
| Method | `POST` |
| Path | `/paper-download/download` |
| 说明 | 先执行 resolve，若状态为 `ready_download`，则下载 PDF |

#### Request Body

继承 `/paper-download/resolve` 的全部字段，并增加：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `output_dir` | string/null | 否 | 相对于后端配置 `paper_download.output_dir` 的子目录 |
| `overwrite` | boolean/null | 否 | 是否覆盖已有文件 |

`output_dir` 安全约束：

- 只能是相对路径。
- 禁止绝对路径，例如 `C:/tmp/papers`。
- 禁止 Windows drive-relative 路径，例如 `C:tmp`。
- 禁止父级目录穿越，例如 `../outside`。
- 最终路径必须位于配置的 `paper_download.output_dir` 下。

#### 请求示例

```json
{
  "name": "LoRA Low-Rank Adaptation of Large Language Models",
  "output_dir": "daily",
  "overwrite": false
}
```

#### Response Body

| 字段 | 类型 | 说明 |
|------|------|------|
| `resolution` | object | `/paper-download/resolve` 的响应体 |
| `download_status` | string | 下载状态，例如 `downloaded`、`not_ready` |
| `file_path` | string/null | 成功下载后的本地路径 |
| `detail` | string | 细节说明 |
| `error_code` | string | 错误码 |

#### 响应示例

```json
{
  "resolution": {
    "raw_input": "LoRA Low-Rank Adaptation of Large Language Models",
    "title": "LoRA: Low-Rank Adaptation of Large Language Models",
    "year": "2021",
    "venue": "arXiv",
    "doi": "10.48550/arXiv.2106.09685",
    "resolve_method": "direct",
    "source": "arxiv",
    "status": "ready_download",
    "pdf_url": "https://arxiv.org/pdf/2106.09685",
    "landing_url": "https://arxiv.org/abs/2106.09685",
    "final_url": "https://arxiv.org/pdf/2106.09685",
    "http_status": "200",
    "content_type": "application/pdf",
    "detail": "",
    "error_code": "",
    "metadata_source": "arxiv",
    "metadata_confidence": "high",
    "suggested_filename": "LoRA Low-Rank Adaptation of Large Language Models__2021.pdf",
    "target_path": "data/papers/daily/LoRA Low-Rank Adaptation of Large Language Models__2021.pdf",
    "probe_trace": []
  },
  "download_status": "downloaded",
  "file_path": "data/papers/daily/LoRA Low-Rank Adaptation of Large Language Models__2021.pdf",
  "detail": "",
  "error_code": ""
}
```

## 16. 安全与数据边界

- API 不应返回任何 API key、token、password 或 `.env` 原始内容。
- 文件写入必须限制在后端配置的数据目录下。
- 自动生成内容不得覆盖人工编辑内容。
- 删除论文、课题、数据集时默认逻辑删除，并保留引用关系复核入口。
- 外部服务错误必须转换为可读错误，避免泄露内部堆栈和敏感路径。
