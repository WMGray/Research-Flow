# 接口规范

## 1. 范围说明

本文档定义 Research-Flow 后端 API 的资源边界与接口契约，当前仅覆盖后端能力，不展开前端页面需求。

接口设计围绕以下资源组织：

- `Paper`
- `Project`
- `Presentation`
- `Dataset`
- `Knowledge`
- `Job`
- `AgentProfile`
- `SkillPackage`
- `SkillBinding`

---

## 2. 通用约定

### 2.1 基础约定

- API 前缀：`/api/v1`
- 数据格式：`application/json`
- 字段命名：`snake_case`
- 时间格式：ISO 8601

### 2.2 返回体格式

```json
{
  "code": "OK",
  "message": "success",
  "data": {},
  "request_id": "req_xxx"
}
```

### 2.3 错误返回格式

```json
{
  "code": "PAPER_NOT_FOUND",
  "message": "paper not found",
  "details": {},
  "request_id": "req_xxx"
}
```

### 2.4 分页约定

列表接口统一支持：

- `page`
- `page_size`
- `order_by`
- `order`

分页返回建议包含：

```json
{
  "items": [],
  "total": 0,
  "page": 1,
  "page_size": 20
}
```

### 3.1 当前实现状态

本文件包含目标 API 规划；当前已挂载路由以 `backend/app/api/` 与测试用例为准。本节只保留截至 2026-04-28 的关键实现差异：

- 已实现或部分实现的主线：`Paper`、`Project`、`Job`、`/paper-download/*`、`/services/mcp*`。
- 文档列出的 `Category`、`/api/v1/config/*`、`Presentation`、`Dataset`、`Knowledge`、Daily Feed、Recommendation、`Zotero`、`Graph`、`Conference` 目标接口尚未挂载。
- Project 已支持删除、模块级任务接口和本地确定性 managed block 写回；真实 LLM 生成、Knowledge / Dataset / Presentation 关联仍待后续接入。
- 当前 `/api/v1/papers/{paper_id}/download` 支持本地开发 fallback；真实 gPaper 下载需要显式启用联网开关和 provider 配置。

### 3.2 当前 envelope 差异

目标响应格式计划为：

```json
{
  "code": "OK",
  "message": "success",
  "data": {},
  "request_id": "req_xxx"
}
```

当前 `Paper`、`Project`、`Job` API 实际使用的 envelope 为：

```json
{
  "data": {},
  "meta": {},
  "error": null
}
```

当前错误响应多由 `HTTPException` 返回，例如：

```json
{
  "detail": {
    "code": "PAPER_NOT_FOUND",
    "message": "Paper not found: 1"
  }
}
```

后续需要在代码或文档中二选一统一：要么把实现迁移到目标 envelope，要么把接口规范收敛到当前 envelope。

| 方法 | 路径 | 说明 |

| `POST` | `/api/v1/papers/{paper_id}/submit-review` | 将优化解析结果置为人工审查中 |
| `POST` | `/api/v1/papers/{paper_id}/confirm-review` | 确认人工审查完成并继续后续链路 |
| `POST` | `/api/v1/papers/{paper_id}/split-sections` | 创建章节切分任务，生成四类 canonical sections |
| `POST` | `/api/v1/papers/{paper_id}/generate-note` | 创建 `note.md` 生成任务 |
| `POST` | `/api/v1/papers/{paper_id}/pipeline` | 串行执行下载、解析、refine、分节、总结的本地闭环 |
| `POST` | `/api/v1/papers/{paper_id}/extract-knowledge` | 创建 Knowledge 提炼任务，默认仅保留置信度 `>= 0.6` 的条目 |
| `POST` | `/api/v1/papers/{paper_id}/extract-datasets` | 创建 Dataset 提炼任务，基于论文 mention 归一化到聚合 Dataset |

**`note.md` 更新请求关键字段**

- `base_version`

**`refined.md` 更新请求关键字段**

- `skill_key`
- `instruction`

**`pipeline` 请求关键字段**

- `download_pdf`
- `parse`
- `refine_parse`
- `split_sections`
- `generate_note`

- `parse` 只负责生成 `parsed/raw.md`

**论文处理链路**

1. `create paper -> metadata_ready`
2. `download`
3. `parse`
4. `refine-parse`
5. `submit-review`
6. `confirm-review`
7. `split sections`
8. `generate-note`
9. `extract-knowledge`

#### 课题任务

以下接口为 M2 目标接口。当前本地实现会同步执行任务、创建 `Job` 并写回对应课题模块文档；后续可替换为真正的异步队列，接口契约保持不变。

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/v1/projects/{project_id}/refresh-overview` | 刷新 overview 统计（非 LLM） |
| `POST` | `/api/v1/projects/{project_id}/generate-related-work` | 整理 related work |
| `POST` | `/api/v1/projects/{project_id}/generate-method` | 生成 method 草稿 |
| `POST` | `/api/v1/projects/{project_id}/generate-experiment` | 生成实验规划建议 |
| `POST` | `/api/v1/projects/{project_id}/generate-conclusion` | 汇总阶段结论 |
| `POST` | `/api/v1/projects/{project_id}/generate-manuscript` | 编译 manuscript 草稿 |

**请求体**（六个接口共用）：

```json
{
  "focus_instructions": "重点关注 2025 年以后的对比方法",
  "included_paper_ids": [123],
  "included_knowledge_ids": [456],
  "skip_locked_blocks": true
}
```

| 字段 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `focus_instructions` | `string` | `""` | 用户附加的针对性生成说明 |
| `included_paper_ids` | `number[]` | 全部已关联论文 | 限定本次输入的论文范围 |
| `included_knowledge_ids` | `number[]` | 全部已关联 Knowledge | 限定本次输入的 Knowledge 范围 |
| `skip_locked_blocks` | `boolean` | `true` | 跳过 `managed="false"` 的已锁定区块 |

**返回体**：

```json
{
  "data": {
    "job_id": "job_xxx",
    "type": "project_generate_related_work",
    "status": "succeeded",
    "progress": 1,
    "message": "Generated related work synthesis.",
    "resource_type": "Project",
    "resource_id": 123,
    "created_at": "2026-04-28T10:00:00Z",
    "updated_at": "2026-04-28T10:00:00Z",
    "result": {
      "output_doc_role": "related_work",
      "managed_blocks": ["related_work_summary", "paper_grouping", "method_comparison"]
    },
    "error": null
  },
  "meta": {},
  "error": null
}
```

任务 `type` 建议取值：

- `project_refresh_overview`
- `project_generate_related_work`
- `project_generate_method`
- `project_generate_experiment`
- `project_generate_conclusion`
- `project_generate_manuscript`

## 7. 错误码建议

| 错误码 | 说明 |
|--------|------|
| `PAPER_NOT_FOUND` | 论文不存在 |
| `PROJECT_NOT_FOUND` | 课题不存在 |
| `PROJECT_DOCUMENT_NOT_FOUND` | 课题文档不存在 |

| `DUPLICATE_DOI` | DOI 重复 |
| `DOCUMENT_VERSION_CONFLICT` | 文档版本冲突 |
| `INVALID_REQUEST` | 参数非法 |
