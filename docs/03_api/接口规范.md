# Paper 全流程接口规范

## 1. 基础约定

- Base URL: `http://127.0.0.1:8000`
- 数据格式：`application/json`
- 当前 API 前缀：`/api`
- 当前不启用 `/api/v1`，等接口稳定后再版本化。
- 当前不引入数据库分页协议，列表接口先返回文件系统扫描结果。

统一响应：

```json
{
  "ok": true,
  "data": {},
  "error": null
}
```

错误响应：

```json
{
  "ok": false,
  "data": null,
  "error": {
    "code": "PAPER_NOT_FOUND",
    "message": "Paper not found"
  }
}
```

## 2. 当前已实现接口

### Health

- `GET /health`

### Dashboard

- `GET /api/dashboard/home`
- `GET /api/dashboard/papers-overview`
- `GET /api/dashboard/discover`
- `GET /api/dashboard/acquire`
- `GET /api/dashboard/library`

### Papers

- `GET /api/papers`
- `GET /api/papers/{paper_id}`
- `POST /api/papers/ingest`
- `POST /api/papers/migrate`
- `POST /api/papers/generate-note`

## 3. 下一步接口

### Config

- `GET /api/config`

返回当前数据根、路径健康、parser 可用性和后续可配置项。

### Paper Parser

- `POST /api/papers/{paper_id}/parse-pdf`
- `GET /api/papers/{paper_id}/parser-runs`

### Paper Actions

- `POST /api/papers/{paper_id}/mark-review`
- `POST /api/papers/{paper_id}/mark-processed`
- `POST /api/papers/{paper_id}/refresh`

这些接口用于人工确认和状态刷新，不接入 LLM 自动判断。

## 4. 请求模型

### `POST /api/papers/ingest`

```json
{
  "source": "C:/path/to/paper-dir",
  "domain": "Speech",
  "area": "Voice-Synthesis",
  "topic": "Singing-Voice-Synthesis",
  "target_path": null,
  "move": false
}
```

### `POST /api/papers/migrate`

与 `ingest` 相同，但服务端按移动模式执行。

### `POST /api/papers/generate-note`

```json
{
  "title": "Example Paper",
  "year": 2026,
  "venue": "ICLR 2026",
  "doi": "",
  "domain": "Speech",
  "area": "Voice-Synthesis",
  "topic": "Flow-Matching",
  "tags": ["paper"]
}
```

### `POST /api/papers/{paper_id}/parse-pdf`

```json
{
  "force": false,
  "parser": "default",
  "options": {}
}
```

### `GET /api/config`

无请求体。

## 6. 状态与错误码

Paper 状态：

- `curated`
- `needs-pdf`
- `parse-pending`
- `parse-failed`
- `needs-review`
- `processed`
- `failed`

建议错误码：

- `PAPER_NOT_FOUND`
- `SOURCE_NOT_FOUND`
- `TARGET_EXISTS`
- `PDF_NOT_FOUND`
- `PDF_PARSE_FAILED`
- `INVALID_METADATA`
- `CONFIG_PATH_INVALID`

## 7. 当前边界

- 暂无数据库分页查询协议
- 暂无异步任务系统接口
- 暂无 LLM Provider 配置接口
- 暂无 Project / Dataset / Knowledge API
