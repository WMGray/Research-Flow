# Research-Flow 文档中心

## 1. 文档分工

| 文件 | 作用 |
|---|---|
| `01_需求与范围.md` | 说明产品要解决什么问题、MVP 做到哪一步、哪些先不做 |
| `02_架构与流程.md` | 说明系统架构、数据模型、页面信息架构和任务流转 |
| `03_API_与实施.md` | 说明接口约定、核心 API、验收口径和实施顺序 |

## 2. 写作原则

1. 先定义边界，再定义功能。
2. 所有流程都要能落到状态、接口和文件路径。
3. 自动化只给建议，关键动作保留人工确认。
4. 本地优先，路径迁移和导出必须可控。
5. 旧草案不再拆成独立文件，统一并回主文档。

## 3. 阅读顺序

1. `01_需求与范围.md`
2. `02_架构与流程.md`
3. `03_API_与实施.md`

## 4. 术语约定

| 术语 | 说明 |
|---|---|
| Paper | 一篇正式或准正式论文 |
| Search Batch | 一次检索任务产生的候选集合 |
| Candidate | 待筛选的候选论文 |
| Curated | 通过 Gate 1 的候选论文队列 |
| Library | 已进入正式库的论文集合 |
| Gate 1 | 候选筛选，决定 keep / reject / pending |
| Gate 2 | 分类与入库审核，决定 accept / modify / reject |

## 5. 存储说明

当前阶段使用本地数据库 + JSON 作为主存储方式，后续再逐步过渡到更完整的数据库表结构。
