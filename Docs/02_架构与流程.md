# Research-Flow 架构与流程

## 总体架构

```text
frontend/
  React + Vite
  HomePage + Router shell

backend/
  FastAPI
  PaperLibrary service
  paper_library CLI

data/
  Discover/
  Acquire/
  Library/
  templates/
```

## 目录说明

### Discover

```text
data/Discover/search_batches/<batch_id>/
  candidates.json
  review.md
```

### Acquire

```text
data/Acquire/curated/<slug>/
  paper.pdf
  note.md
  metadata.yaml
```

### Library

```text
data/Library/<Domain>/<Area>/<Topic>/<slug>/
  paper.pdf
  note.md
  refined.md
  metadata.yaml
```

未分类时进入：

```text
data/Library/unclassified/<slug>/
```

## 状态流转

```text
search-batch -> curated -> processed
                    \-> needs-pdf
                    \-> needs-review
                    \-> failed
```

## 后端职责

1. 读取 `data/` 并聚合首页数据。
2. 解析 `metadata.yaml` 与 `note.md` front matter。
3. 提供 ingest / migrate / note-template 能力。
4. 保证已有目标目录不被静默覆盖。

## 前端职责

1. 渲染 HomePage 总入口。
2. 展示统计卡片、近期论文、待处理列表、Dashboard 跳转。
3. 对接真实后端接口，不内置静态 mock。
