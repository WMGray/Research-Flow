# Research-Flow 需求与范围

## 背景

本项目聚焦 Paper 工作流 MVP。目标是把 `05_Research` 中已经验证过的文献处理思路，收敛成一个真正可运行的前后端项目。

## MVP 目标

1. 建立前后端分离项目骨架。
2. 建立本地文献库真源 `data/`。
3. 支持 Search Batch、Curated、Library 三层数据视图。
4. 支持 Paper ingest / migrate / note-template 脚本。
5. 提供 HomePage 汇总接口和页面展示。

## 本轮范围

### 后端

1. `GET /api/dashboard/home`
2. `GET /api/dashboard/papers-overview`
3. `GET /api/dashboard/discover`
4. `GET /api/dashboard/acquire`
5. `GET /api/dashboard/library`
6. `GET /api/papers`
7. `GET /api/papers/{paper_id}`
8. `POST /api/papers/ingest`
9. `POST /api/papers/migrate`
10. `POST /api/papers/generate-note`

### 前端

1. HomePage
2. 基础 Sidebar / TopBar / Router 壳
3. Discover / Acquire / Library / Runtime / Logs 占位页

### 数据

1. `data/Discover/search_batches/`
2. `data/Acquire/curated/`
3. `data/Library/`
4. `data/templates/`

## 不做

1. skills 自动调用
2. Celery / Redis
3. LLM 自动分类
4. 真正 PDF 解析
5. Project / Dataset / Knowledge 的完整功能
