# Research-Flow 需求与范围

## 1. 项目背景

Research-Flow 面向科研人员的论文与课题管理场景，目标是把论文发现、筛选、下载、解析、分类、阅读笔记、课题关联和 Dashboard 管理整合到一个本地优先的研究工作台中。

## 2. 核心问题

1. 论文从发现到入库缺少统一状态。
2. PDF、metadata、解析结果和笔记容易脱节。
3. 候选论文与正式论文库混在一起。
4. 自动分类和笔记生成需要人工确认。
5. Dashboard 只看得到数据，难以看见流程。

## 3. MVP 目标

1. 跑通论文闭环。
2. 支持 Search Batch 和 Candidate 筛选。
3. 支持 PDF 绑定、解析和分类审核。
4. 支持正式库、阅读笔记和 Obsidian 导出。
5. 支持 Dashboard 总览和任务追踪。
6. 当前实现先用本地数据库 + JSON，后续再过渡到正式数据库结构。

## 4. 非目标

1. 不做完整 Zotero 替代。
2. 不做复杂知识图谱推理。
3. 不做团队协作和多用户权限。
4. 不做完整桌面封装。
5. 不自动绕过付费墙获取论文。

## 5. 功能范围

### 5.1 文献管理

1. 检索论文并生成 Search Batch。
2. 维护 Candidate 的 keep / reject / pending。
3. 支持 Gate 1 把 keep 条目转为 Curated。
4. 绑定本地 PDF 并记录 metadata。
5. 生成 refined.md、note.md 和导出目录。

### 5.2 课题管理

1. 创建 Project。
2. 关联相关论文。
3. 记录 Related Work、Method、Experiment、Conclusion。
4. 支持从论文沉淀研究线索。

### 5.3 任务与导出

1. 任务队列可恢复。
2. 解析、分类、笔记生成、导出都走统一任务。
3. 导出到 Obsidian 目录时不覆盖用户手写内容。

## 6. 优先级

### P0

1. Search Batch。
2. Candidate 筛选。
3. PDF 绑定与解析。
4. Gate 2 审核。
5. 正式库与 Dashboard。

### P1

1. 阅读笔记生成。
2. Project 关联。
3. Obsidian 导出。
4. 任务重试与日志。

### P2

1. 论文推荐。
2. 知识图谱。
3. 投稿与 Rebuttal 辅助。
